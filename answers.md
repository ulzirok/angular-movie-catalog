# Ответы на вопросы

## Как вы можете описать процесс распространения событий в JavaScript (Event Propagation)? Расскажите о его основных фазах. Какие есть способы применения знаний о фазах рааспространения событий на практике?

В JavaScript события распространяются по DOM-дереву по определённым правилам. Этот процесс называется Event Propagation. Он описывает, как событие проходит через элементы страницы.
Существует 3 фазы распространения события:
1. Capture phase (фаза перехвата)
Событие начинается от корневого элемента (window, document) и идет вниз по DOM-дереву к целевому элементу. На этой фазе событие еще не достигло элемента, по которому был клик.
2. Target phase (целевая фаза)
Событие достигает элемента, на котором произошло действие (например, кнопка).
3. Bubbling phase (фаза всплытия)
После этого событие начинает подниматься вверх по DOM-дереву от целевого элемента к document. По умолчанию обработчики событий в JavaScript работают на фазе всплытия.
На практике используется:
- для делегирования событий (один обработчик на родителе вместо множества на дочерних элементах)
- для предотвращения лишних обработчиков с помощью stopPropagation.

## Расскажите, что такое Promise в JavaScript, как он работает, какие основные состояния у него бывают? Каким еще способомможно обработать асинхронный код? Какую роль в асинхронности играет событийный цикл (Event Loop)?

Promise - это объект в JavaScript, который представляет собой результат асинхронной операции, которая может выполниться позже или завершиться с ошибкой.
У Promise есть 3 состояния:
1. pending - ожидание выполнения,
2. fulfilled - успешно выполнен,
3. rejected - выполнен с ошибкой.
Promise создается, например, при работе с HTTP-запросами. Результат промиса можно обработать через методы:
- then() при успешном выполнении,
- catch() при ошибке,
- finally() выполняется всегда.
Кроме Promise, асинхронный код можно обрабатывать с помощью:
- callbacks (устаревший и менее удобный способ)
- async / await - синтаксический сахар над Promise, делает код более читаемым.
Асинхронность в JavaScript обеспечивается событийным циклом (Event Loop).
Event Loop следит за:
- Call Stack (стек вызовов)
- Web APIs
- очередями задач (macrotasks и microtasks)
Когда стек вызовов освобождается, Event Loop помещает готовые асинхронные задачи обратно в стек для выполнения. Благодаря этому JavaScript, будучи однопоточный, может выполнять асинхронные операции.

## Что такое ООП, и какие его ключевые принципы вы могли бы описать? При объяснении приводите примеры. Максимально подробно расскажите о том, как ООП реализовано в JavaScript?

ООП (объектно-ориентированное программирование) - это подход к разработке, при котором программа строится на основе объектов, которые объединяют данные и логику. В JavaScript ООП основано на прототипной модели. Даже при использовании class под капотом работает прототипное наследование. Классы в JavaScript - это синтаксический сахар над прототипами.
Основные принципы ООП:
1. Инкапсуляция - это когда мы прячем внутреннюю логику объекта и даём работать с ним только через понятные методы, не показывая, как всё устроено внутри. В JavaScript это реализуется через замыкания, private поля (через #), getter и setter.
2. Наследование - один объект или класс может наследовать свойства и методы другого. В JavaScript это реализовано через прототипы и ключевое слово extends.
3. Полиморфизм - возможность использовать один метод для разных реализаций. Например, переопределение методов в дочерних классах.
4. Абстракция - выделение общей логики и скрытие деталей реализации. То есть, это способ упростить работу с кодом, когда мы используем объект или метод, не вникая в его внутреннюю реализацию. Это помогает писать более понятный и поддерживаемый код. В JavaScript это достигается через абстрактные классы.

## Опишите максимально детально, как браузер обрабатывает URL, начиная с момента его ввода в адресную строку и заканчивая отображением страницы. Какие технологии позволяют ускорить процесс? Какие проблемы (безопастность, междоменное взаимодействие) могут возникать в процессе?

Когда пользователь вводит URL в адресную строку, браузер начинает процесс загрузки страницы. 
Сначала браузер анализирует, по какому адресу и по какому протоколу нужно обращаться (http или https). 
После этого браузер узнаёт IP-адрес сервера, на котором находится сайт, используя DNS или данные из кэша.
Затем браузер устанавливает соединение с сервером и отправляет HTTP-запрос, чтобы получить страницу.

Когда сервер отвечает, браузер:
- получает HTML-файл
- на основе HTML строит структуру страницы
- загружает стили и применяет их
- после этого отображает страницу пользователю

Чтобы страницы загружались быстрее, браузеры и серверы используют:
- кэширование (повторная загрузка без запроса к серверу)
- сжатие данных (gzip, brotli)
- современные протоколы (HTTP/2, HTTP/3)
- отложенную загрузку изображений (lazy loading)
- CDN для хранения ресурсов ближе к пользователю

Во время загрузки страницы могут возникать проблемы, например:
- ограничения на запросы между разными доменами (CORS)
- уязвимости безопасности, такие как XSS.
